# Beginner Guide: How to Use Rkia

## What you are actually building

You are not training a giant model from scratch.
You are building an AI operating system.

That system has 4 main parts:

1. Brain
   - the OpenAI model
2. Personalities
   - Rkia and the advisors
3. Tools
   - memory, leads, invoices, deployment, research, etc.
4. Rules
   - when to act automatically, when to ask you first

## What to do first

Do not start with apps, websites, videos, invoices, and legal tools all at once.

Start with this exact order:

### Step 1
Get the local console running.

### Step 2
Use it for simple conversations with each advisor.

### Step 3
Save 5 to 10 memory notes so the system starts feeling personal.

### Step 4
Save some fake leads so you can see how business memory works.

### Step 5
Pick one real mission:
- HY Services relaunch
- HYFACTS007 workflow
- one app idea

Only after that should you add more tools.

## How to think like a builder

When Rkia feels weak, it usually means one of these is missing:
- memory
- a tool
- a clear prompt
- a safe approval rule

Do not say “the AI is bad” too quickly.
Usually the system design is the problem, not the model.

## What “good” looks like

A good first version means:
- you can talk to each advisor
- it remembers notes
- it stores leads
- it gives useful answers
- you understand how to run it

That is enough for version 1.
